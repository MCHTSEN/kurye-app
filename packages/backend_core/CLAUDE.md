# backend_core

Shared interfaces, domain models, and logging infrastructure for all backend implementations.

## Purpose

This package defines the contracts that every backend (Firebase, Supabase, Custom API) must implement. It contains zero external SDK dependencies — only pure Dart code and the `logger` package for logging.

## File Structure

```
lib/
  backend_core.dart              → Barrel export
  src/
    analytics_service.dart       → Analytics tracking interface
    api_client.dart              → REST API client interface
    auth_gateway.dart            → Authentication gateway interface
    auth_repository.dart         → Auth repository interface (higher level)
    auth_repository_impl.dart    → Shared auth repository with analytics tracking
    backend_module.dart          → Abstract factory for backend services
    credit_access_service.dart   → Credit check interface
    token_refresh_service.dart   → Token refresh interface
    domain/
      analytics_event.dart       → Analytics event model
      auth_session.dart          → Auth session model
      auth_user.dart             → Auth user model
    logging/
      app_log_config.dart        → LogTag enum + per-tag toggle config
      app_logger.dart            → Logger wrapper with tag-based filtering
```

## Key Patterns

### BackendModule

Abstract factory that each backend implements:

```dart
abstract class BackendModule {
  Future<void> initialize();
  AuthGateway createAuthGateway();
  TokenRefreshService createTokenRefreshService();
  CreditAccessService? createCreditAccessService();
}
```

### Logging

Every implementation class should include:

```dart
static final _log = AppLogger('ClassName', tag: LogTag.auth);
```

Log levels: `d` (debug), `i` (info), `w` (warning), `e` (error with optional error/stackTrace).

Tags control which logs are visible via `logConfig`.

## Adding a New Backend Capability

1. Add the abstract method to the relevant interface (e.g., `AuthGateway`)
2. Update `AuthRepositoryImpl` if the method needs analytics tracking
3. Each backend package must implement the new method
4. Add tests for the new capability

## Adding a New Backend Package

1. Create `packages/backend_<name>/` with `pubspec.yaml` depending on `backend_core`
2. Implement `BackendModule` subclass
3. Implement all required interfaces (`AuthGateway`, `TokenRefreshService`)
4. Create barrel export file
5. Add entry point `lib/main_<name>.dart` in the main app
6. Add to workspace in root `pubspec.yaml`
7. Write CLAUDE.md for the new package

## Rules

- NO external SDK dependencies (only `logger`)
- All classes are abstract or pure Dart implementations
- Every implementation should use AppLogger for observability
