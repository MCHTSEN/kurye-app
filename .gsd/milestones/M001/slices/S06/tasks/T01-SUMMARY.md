# T01: Add getHistory() to SiparisRepository with server-side filtering

**Added `getHistory()` to SiparisRepository contract with 5 optional filter params, implemented in Supabase (server-side PostgREST chaining) and fake (client-side), plus `siparisHistoryProvider` Riverpod family.**

## What Happened

1. Added `getHistory()` abstract method to `SiparisRepository` with optional named params: `startDate`, `endDate`, `musteriId`, `cikisId`, `ugramaId`. Returns `Future<List<Siparis>>`.
2. Implemented in `SupabaseSiparisRepository`: starts with `.inFilter('durum', [tamamlandi, iptal])`, chains `.gte`/`.lte` for date range and `.eq` for each non-null filter, orders by `created_at` descending. Logged with `_log.i()`.
3. Implemented in `FakeSiparisRepository`: filters `store.values` client-side with equivalent logic. Null `createdAt` entries are excluded from date-range filtering. Sorted descending by `createdAt`.
4. Added `siparisHistory` provider in `siparis_providers.dart`. Code-gen produced the family provider with all 5 optional params.
